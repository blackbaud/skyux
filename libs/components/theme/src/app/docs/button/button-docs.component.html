The button classes create buttons to trigger actions from within an interface.

The primary button triggers the most important or most frequently used action on a page or the progressive terminal action on a modal such as Save. Pages and modals should only include one primary button to ensure that it draws user attention. For more information, see the primary action guidelines.

class="sky-btn sky-btn-primary"

Secondary buttons trigger less important or less frequently used actions. SKY UX reduces the visual weight to make them less attention-drawing than the primary buttons.

class="sky-btn sky-btn-default"

Link buttons trigger actions that take users to other pages or actions that end tasks or exit modals (such as Cancel).

class="sky-btn sky-btn-link"

Inline link buttons trigger the same type of actions as link buttons, but the styling is altered to fit inline.

class="sky-btn sky-btn-link-inline"

Icon buttons trigger common actions, such as edit and delete, when space is too tight for text labels and icons are sufficient to represent actions. For example, use icon buttons to invoke inline forms from repeater rows.

class="sky-btn sky-btn-icon-borderless"

The disable class disables all button types. The most common use case is to disable actions until users complete prerequisite tasks.

class="sky-btn-disabled"

For buttons with multiple possible direct objects, such as Edit goals or Email contact buttons in page headers, use the following format for button text:

<Verb> <direct object>

Additional button text guidelines:

  • Use sentence-style capitalization.
  • Use singular or plural direct objects to indicate whether the button acts on one or multiple items.
  • Don't include articles. For example, use Delete goal rather than Delete a goal.
  • Don’t append extra words such as “details.” For example, use Edit goal rather than Edit goal details.
