The icon component displays a Font Awesome icon or a custom SKY UX icon.

Icons are pictorial representations of objects. They help define the visual identity of SKY UX applications, and they provide a shorthand to visually convey meaning that users can understand at a glance. SKY UX uses glyphic, mono-color icons to facilitate, guide, and inform users.

Icons should not be used solely for decorative purposes.

To determine whether an icon is available for a new component, check the SKY UX icon dictionary below. Then make sure that your use case is consistent with how other components use the icon.

  • Do not use icons with established meanings for different purposes.

  • Be mindful of how icons are used in specific components such as grids, tiles, buttons, and modals

Do not use icons with established meanings for different purposes.

Icons usually inherit their size and color from parent elements, particularly when you use them inline. If you use icons in a non-inline context and need a different size, Font Awesome provides size multiplier classes. Use the smallest size multiple that accomplishes your goals.

The icon dictionary provides a complete list of the Font Awesome icons that we recommend for use in SKY UX. We support Font Awesome version 4.7.0. If Font Awesome does not have a particular icon, it is probably not common enough for its purpose to be immediately obvious to users.

SKY UX also provides a set of custom icons.
